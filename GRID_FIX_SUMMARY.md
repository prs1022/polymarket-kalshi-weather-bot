# 网格订单价格修复说明

## 🐛 问题描述

**你发现的现象：**
- 市场：btc-updown-5m-1784105400
- 订单簿：买1=53¢，卖1=54¢
- 实际挂单：34¢ (5份额)，20¢ (5份额)
- **问题：34¢ 远低于市场价 53¢，几乎不可能成交！**

## 🔍 根本原因

### 1. 固定下限太低
```python
GRID_LOWER_BOUND = 0.20  # 20美分 - 固定值
```

### 2. 斐波那契分布导致网格偏离过大
当前价 49¢ → 下限 20¢ = **29¢ 跨度 (59%)**

```
Level 0: 49¢ - 25% × 29¢ = 42¢ → 34¢ (实际)
Level 1: 49¢ - 50% × 29¢ = 34¢ → 20¢ (实际)
Level 2: 49¢ - 100% × 29¢ = 20¢
```

**需要价格下跌 31% 才能成交第一层！**

### 3. 为什么是 DOWN 方向但记录 49¢？

- 市场 UP token 价格：50¢
- 市场 DOWN token 价格：50¢ (1 - 0.50)
- 方向：DOWN（做空 BTC）
- 挂单买入：DOWN token @ 49¢ → 34¢

## ✅ 解决方案

### 修改 `backend/core/grid.py`

**旧代码：**
```python
if extreme_price is None:
    extreme_price = _get_extreme_price()  # 固定 0.20
```

**新代码：**
```python
if extreme_price is None:
    # 动态下限：当前价的85%（下跌15%范围内分批买入）
    extreme_price = max(current_price * 0.85, settings.GRID_LOWER_BOUND)
```

### 效果对比

| 指标 | 修复前 | 修复后 | 改进 |
|-----|--------|--------|------|
| **网格范围** | 29¢ (59%) | 7¢ (15%) | ✅ 缩小 4倍 |
| **Level 0** | 42¢ | 47¢ | ✅ +5¢ |
| **Level 1** | 34¢ | 45¢ | ✅ +11¢ |
| **Level 2** | 20¢ | 42¢ | ✅ +22¢ |
| **成交概率** | 极低 | 高 | ✅ 大幅提升 |

### 新网格示例

当前价 49¢，动态下限 42¢ (49¢ × 0.85)：

```

Level 1: 45¢ - 距当前仅 8% ✅
```

## 🎯 策略说明

### 网格交易原理
分批买入，降低平均成本：
- 价格下跌 → 更低价格买入 → 平均成本降低
- 价格反弹 → 止盈/止损

### 合理的网格范围
- **过大**（59%）：永远不成交 ❌
- **过小**（<5%）：失去分批优势 ⚠️
- **适中**（15%）：平衡成交率和成本优化 ✅

### 动态下限优势
1. **跟随市场**：价格高时范围高，价格低时范围低
2. **避免陷阱**：不会在 50¢ 时挂 20¢ 单
3. **提高效率**：资金不会长时间闲置
4. **保留策略**：仍然是"下跌时分批买入"，只是范围合理化
