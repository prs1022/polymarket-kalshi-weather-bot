# 网格配置变更说明

## 📝 变更内容

### 修改文件
1. `backend/config.py`
2. `backend/core/grid.py`

### 配置变更

| 配置项 | 旧值 | 新值 | 说明 |
|-------|------|------|------|
| `GRID_LEVELS` | 2 | **3** | 网格层数增加到3层 |
| `GRID_MODE` | fibonacci | **equal** | 改为等间距均分 |
| 下限计算 | 动态 (current*0.85) | **固定 (0.20)** | 使用固定20¢作为下限 |

## 🎯 效果对比

### 示例：当前价 45¢

#### 旧配置（Fibonacci + 动态下限）
```
当前价: 45¢
下限: 38¢ (45¢ × 0.85)
范围: 7¢ (15%)

网格分布 (2层):
  Level 0: 42¢ (斐波那契分布)
  Level 1: 40¢

问题: ❌ 间隔太小，需要多层才能覆盖更大范围
```

#### 新配置（Equal + 固定下限）
```
当前价: 45¢
下限: 20¢ (固定)
范围: 25¢ (56%)

网格分布 (3层):
  Level 0: 37¢ (均分 1/3)
  Level 1: 28¢ (均分 2/3)
  Level 2: 20¢ (下限)

优势: ✅ 大跨度，低价抄底
```

#### 新配置（2层模式）
```
当前价: 45¢
下限: 20¢
范围: 25¢

网格分布 (2层):
  Level 0: 32¢ (均分 1/2)
  Level 1: 20¢ (下限)

符合需求: ✅ 45¢ → 32¢ → 20¢
```

## 📊 实际案例对比

### 场景1: DOWN token @ 45¢

**旧配置:**
```
挂单: 42¢, 40¢
需要下跌: 3¢ (7%) 才成交第1层
成交速度: 快 ✅
平均成本: 高 ⚠️
```

**新配置 (3层):**
```
挂单: 37¢, 28¢, 20¢
需要下跌: 8¢ (18%) 才成交第1层
成交速度: 慢 ⚠️
平均成本: 低 ✅（如果全部成交）
```

### 场景2: 不同市场价格

| 当前价 | 旧配置网格 | 新配置网格 (3层) | 跨度对比 |
|-------|-----------|----------------|---------|
| 50¢ | 47¢, 44¢ | 40¢, 30¢, 20¢ | 6¢ → 30¢ |
| 45¢ | 42¢, 40¢ | 37¢, 28¢, 20¢ | 5¢ → 25¢ |
| 40¢ | 37¢, 35¢ | 33¢, 27¢, 20¢ | 5¢ → 20¢ |
| 30¢ | 28¢, 26¢ | 27¢, 23¢, 20¢ | 4¢ → 10¢ |

## ⚖️ 优劣分析

### 新配置优势 ✅

1. **更大跨度**
   - 从 7¢ 扩大到 25¢
   - 覆盖更大的价格范围

2. **低价抄底**
   - 能在 20¢-30¢ 低价区间买入
   - 如果暴跌，能获得更低成本

3. **均匀分布**
   - 等间距易于理解
   - 每层间距一致

4. **符合预期**
   - 完全按照 45→32→20 的需求
   - 逻辑清晰

### 新配置劣势 ⚠️

1. **成交慢**
   - 需要价格下跌 18% 才成交第1层
   - 旧配置只需 7%

2. **资金占用**
   - 订单可能长时间挂着不成交
   - 降低资金使用效率

3. **错过机会**
   - 小幅下跌（5-10%）不会成交
   - 旧配置会捕捉这些机会

4. **极端依赖**
   - 需要价格暴跌才能全部成交
   - 如果只跌 10%，一单都不会成交

## 🎯 适用场景

### 新配置适合：

✅ **预期大幅波动**
- BTC 剧烈波动期
- 重大消息前后
- 市场恐慌时

✅ **抄底策略**
- 相信价格会暴跌
- 愿意等待低价
- 不急于成交

✅ **大本金**
- 可以分散更多资金
- 不担心占用

### 旧配置适合：

✅ **小幅波动**
- BTC 震荡期
- 正常交易时段

✅ **快速成交**
- 希望快速建仓
- 追求资金效率

✅ **小本金**
- $10-20 本金
- 需要快速周转

## 💡 建议

### 方案1: 混合策略（推荐）

根据市场情况切换：
```python
# 波动率 > 2% → 使用新配置（大跨度）
# 波动率 < 1% → 使用旧配置（小跨度）
```

### 方案2: 可调参数

添加配置项：
```python
GRID_USE_DYNAMIC_LOWER: bool = False  # False=固定20¢, True=动态85%
```

### 方案3: 分层策略

- 第1笔交易：小跨度（快速成交）
- 第2-3笔：大跨度（抄底）
