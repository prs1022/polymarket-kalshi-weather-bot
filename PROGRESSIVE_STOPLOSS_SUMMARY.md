# 渐进式止损功能说明

## 📝 功能描述

**旧逻辑：** 等待所有网格层成交后才挂止损单 @ 平均成本价

**新逻辑：** 每成交一层立即更新止损单 @ 平均成本 + 5¢

## 🎯 修改内容

### 1. 配置文件 (`backend/config.py`)

新增配置项：
```python
PROGRESSIVE_STOP_LOSS: bool = True  # 渐进式止损开关
STOP_LOSS_OFFSET: float = 0.05     # 止损加价 5美分（覆盖手续费）
```

### 2. 调度器 (`backend/core/scheduler.py`)

修改 `check_grid_fills_job()` 函数：
- 每次网格订单成交后立即检查
- 计算当前平均成本
- 设置/更新止损价 = 平均成本 + 5¢
- 对模拟盘和实盘分别处理

## 📊 工作流程

### 示例：3层网格 49¢, 34¢, 20¢

```
时刻    事件              平均成本    止损价      说明
──────────────────────────────────────────────────────
T+0     开仓挂单          -           -          等待成交
T+60    Level 0 @ 49¢     49¢         54¢        ✅ 第1次挂止损
T+120   Level 1 @ 34¢     42¢         47¢        ✅ 更新止损
T+180   Level 2 @ 20¢     34¢         39¢        ✅ 再次更新
```

### 计算公式

```python
# 平均成本
avg_cost = sum(price × shares) / sum(shares)

# 止损价
stop_loss = avg_cost + 0.05  # +5美分
```

### 实际计算

```
第1层成交: 49¢ × 5份
平均成本 = 49¢
止损价 = 49¢ + 5¢ = 54¢

第2层成交: 34¢ × 5份
平均成本 = (49×5 + 34×5) / 10 = 41.5¢
止损价 = 41.5¢ + 5¢ = 46.5¢

第3层成交: 20¢ × 5份
平均成本 = (49×5 + 34×5 + 20×5) / 15 = 34.3¢
止损价 = 34.3¢ + 5¢ = 39.3¢
```

## 🆚 新旧逻辑对比

| 对比项 | 旧逻辑 | 新逻辑 |
|-------|--------|--------|
| **触发时机** | 全部成交后 | 每层成交后 |
| **保护时间** | 仅最后阶段 | 全程保护 |
| **止损价** | 平均成本 | 平均成本 + 5¢ |
| **手续费** | 不覆盖 ⚠️ | 覆盖 ✅ |
| **更新频率** | 一次 | 动态更新 |
| **风险** | 前期无保护 ❌ | 持续保护 ✅ |

## 💰 为什么加 5¢？

### Polymarket 费用结构
```
交易手续费: 0% (Maker & Taker)
Gas 费: ~2-3¢ (Polygon 网络)
滑点: ~1-2¢ (市场波动)
总成本: ~3-5¢
```

### 不加价的风险
```
成本: 42¢
止损: 42¢ (平均成本)
卖出: 42¢ - 3¢ (gas+滑点) = 39¢
结果: 亏 3¢ ❌ (名义保本，实际小亏)
```

### 加价的保护
```
成本: 42¢
止损: 47¢ (成本 + 5¢)
卖出: 47¢ - 3¢ (gas+滑点) = 44¢
结果: 赚 2¢ ✅ (真正小盈)
```

## ⚙️ 技术细节

### 止损单更新逻辑

```python
if newly_filled:
    # 1. 更新平均成本和份额
    update_trade_from_grid(trade, all_grid)
    
    # 2. 计算新止损价
    new_stop_loss = trade.entry_price + STOP_LOSS_OFFSET
    
    # 3. 如果新价格更优（更高），则更新
    if old_stop_loss is None or new_stop_loss > old_stop_loss:
        trade.stop_loss_price = new_stop_loss

        # executor.cancel_order(old_order_id)
        # executor.place_limit_sell(...)
```

### 为什么只在价格更高时更新？

```
第1层 @ 49¢ → 止损 @ 54¢
第2层 @ 34¢ → 成本降到 42¢ → 止损应降到 47¢

如果价格已经反弹到 50¢：
  - 止损 @ 54¢ 仍然有效
  - 不应该降到 47¢（会降低保护）
  
所以只在新止损价 > 旧止损价时更新
```

## ✅ 优势总结

1. **实时保护** 
   - 每层成交立即挂止损
   - 不再等待全部成交

2. **覆盖成本**
   - +5¢ 确保盈利覆盖手续费
   - 真正的小盈，不是名义保本

3. **动态调整**
   - 随成本下降而调整
   - 始终保持合理止损位

4. **灵活退出**
   - 价格反弹能快速获利
   - 避免利润回吐

## ⚠️ 注意事项

1. **止损不是万能的**
   - 价格持续下跌不反弹，止损不触发
   - 最终仍按市场实际结果结算

2. **止损价会下降**
   - 随着成本摊低，止损价也降低
   - 这是正常的，因为成本确实降低了

3. **需要价格反弹**
   - 止损单需要价格反弹到止损价才成交
   - 如果一直跌，止损不会触发
